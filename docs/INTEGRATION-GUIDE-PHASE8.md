Antes (Monolito)\n```\nWebSocket Handler\n  └─ BD Queries (no transacción)\n  └─ Lógica mezclada\n  └─ Sin idempotencia\n  └─ Sin validación clara\n💥 Race conditions \n```\n\n### 2. Después (Modular)\n```\nWebSocket Handler\n  └─ Coordinator (valida contexto)\n  └─ Manager (maneja transacción)\n      └─ 0️⃣ Registry.find(actionId)  ← ANTES de lock\n      └─ 1️⃣ sequelize.transaction()\n      └─ 2️⃣ Lock + Reload\n      └─ 3️⃣ Mapper (BD → puro)\n      └─ 4️⃣ RulesEngine.validate()\n      └─ 5️⃣ RulesEngine.execute()\n      └─ 6️⃣ Repository.applyState()\n      └─ 7️⃣ Registry.register()  ← DENTRO de transacción\n✅ Seguro, atómico, idempotente\n```\n\n---\n\n## 📞 Troubleshooting\n\n### Error: \"Acción no encontrada en tabla processed_actions\"\n**Causa:** SQL migration no ejecutada\n**Fix:**\n```bash\npsql -U postgres -d cg_server -f src/migrations/sql/001_create_processed_actions_table.sql\n```\n\n### Error: \"unique constraint violation on action_id\"\n**Causa:** Intentando registrar mismo actionId sin transacción\n**Fix:** Usar mismo actionId es INTENCIONAL (idempotencia). Si ves este error fuera de reintento, revisar logic.\n\n### Error: \"Lock acquisition timeout\"\n**Causa:** Otro usuario tiene lock, esperando > 30 segundos\n**Fix:** Aumentar timeout en Sequelize config, O optimizar queries dentro de transaction\n\n### Error: \"Match not found\"\n**Causa:** match_id inválido o match deleted\n**Fix:** Validar que match_id es correcto UUID\n\n### Respuesta Lenta (> 500ms)\n**Causa:** SERIALIZABLE isolation level es lento\n**Opciones:**\n1. Cambiar a REPEATABLE_READ (menos seguridad)\n2. Optimizar queries dentro de transacción\n3. Usar Redis locking (advanced)\n\n---\n\n## 🎉 Success Criteria\n\nPhase 8 está completo cuando:\n\n1. ✅ Handlers refactorizados funcionan en Postman\n2. ✅ Godot client puede conectar y enviar acciones\n3. ✅ Idempotencia funciona (reintento devuelve cache)\n4. ✅ BD tiene registros en processed_actions\n5. ✅ No hay race conditions (validado con 2 clientes simultáneamente)\n6. ✅ Latencia < 500ms (aceptable con lock)\n7. ✅ Error handling robusto\n8. ✅ Logs informativos\n\nAl completar:\n- [ ] Pasar a Phase 8.3 (Validación Producción)\n- [ ] Deprecate handlers viejos (mantener 2 semanas)\n- [ ] Planificar Phase 9 (Limpieza + Optimizaciones)\n\n---\n\n## 📚 Referencias\n\n- `src/engine/` - Lógica pura (100% determinística)\n- `src/services/coordinators/` - Validación contexto\n- `src/services/game/` - Managers (TurnManager, CardManager, AttackManager)\n- `src/services/mappers/` - Bridge BD ↔ puro\n- `src/services/registries/` - Idempotencia\n- `websocket-integrations.ts` - Nuevos handlers\n- `ARCHITECTURE-REFACTOR.md` - Diseño completo\n- `RESUMEN-IMPLEMENTACION-PHASE7.md` - Detalles Phase 7\n\n
